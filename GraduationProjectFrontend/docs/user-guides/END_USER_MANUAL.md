## Getting Started

### First Login
1. Navigate to the application URL
2. Enter your credentials:
   - **Email**: Your university email
   - **Password**: Initial password (change on first login)
3. Complete your profile setup

### Dashboard Overview
Your dashboard is customized based on your role. All users see:
- **Quick Stats**: Key metrics for your work
- **Recent Activity**: Latest updates
- **Upcoming Events**: Deadlines and meetings
- **Notifications**: Important alerts

## Student Guide

### 1. Joining a Team

**As a Team Member:**
1. Go to **Teams** page
2. Browse available teams or use search
3. Click **Request to Join**
4. Wait for team leader approval

**Creating a Team:**
1. Click **Create Team** button
2. Fill in team details:
   - Team name
   - Project description
   - Technology stack
   - Max members (4-6 recommended)
3. Generate invite code to share with members

### 2. Working with Tasks

**Viewing Your Tasks:**
- Dashboard shows your assigned tasks
- **Tasks** page shows full task board
- Filter by: Status, Priority, Due date

**Updating Tasks:**
1. Click on a task card
2. Update status by dragging or clicking status
3. Add comments for team communication
4. Attach files if needed
5. Mark as complete when done

**Creating Tasks (Team Leaders):**
1. Click **+ Add Task**
2. Enter task details:
   - Title and description
   - Assign to team member
   - Set priority and deadline
   - Add tags for organization
3. Save task

### 3. Calendar & Meetings

**Viewing Events:**
- **Calendar** shows all your events
- Color-coded by type:
  - Blue: Team meetings
  - Purple: Virtual meetings
  - Red: Deadlines
  - Green: Personal events

**Joining Meetings:**
1. Click on meeting in calendar
2. View meeting details
3. Click **Join Meeting** (for virtual)
4. Or note location (for in-person)

**Sync with Google Calendar:**
1. Go to **Settings** → **Calendar**
2. Click **Connect Google Calendar**
3. Authorize access
4. Your events sync automatically

### 4. Using GitHub Integration

**Connecting Repository:**
1. Go to **GitHub** page
2. Click **Connect Repository**
3. Enter GitHub URL
4. Authenticate with GitHub
5. Grant repository access

**Making Commits:**
1. Click **Create Commit**
2. Write commit message (follow conventions)
3. Add detailed description
4. Select branch
5. Commit changes

**Creating Pull Requests:**
1. Click **Create PR**
2. Select source and target branches
3. Write PR title and description
4. Submit for review

### 5. Submitting Deliverables

**Project Proposal:**
1. Go to **Proposals** page
2. Fill in all sections:
   - Problem statement
   - Objectives
   - Scope and limitations
   - Tech stack
   - Timeline
3. Save as draft or submit

**Other Submissions:**
1. Navigate to **Submissions**
2. Select deliverable type
3. Upload files
4. Add comments
5. Submit before deadline

### 6. Communication

**Team Chat:**
- Real-time messaging with team
- @mention teammates
- Share files and links
- React to messages with emojis

**Discussions:**
- Post questions to entire team
- Get answers from teammates or supervisors
- Mark best answer
- Search previous discussions

### 7. Gamification

**Earning XP:**
- Complete tasks: 10-50 XP
- Submit on time: Bonus XP
- Help teammates: 15 XP
- Attend meetings: 5 XP
- Daily streak: 10 XP

**Unlocking Achievements:**
- View available achievements
- Track your progress
- Unlock special badges
- Compete on leaderboard

**Using Coins:**
1. Go to **Shop** (in gamification)
2. Browse items:
   - Custom themes
   - Avatar frames
   - Profile badges
   - Priority support
3. Purchase with earned coins

### 8. Profile Management

**Updating Profile:**
1. Click profile avatar → **Profile**
2. Edit information:
   - Bio and expertise
   - Skills and interests
   - Contact info
   - Privacy settings
3. Upload profile picture
4. Save changes

**Changing Password:**
1. Go to **Settings** → **Security**
2. Click **Change Password**
3. Enter current password
4. Enter new password (min 8 characters)
5. Confirm and save

## Team Leader Guide

### 1. Team Management

**Managing Team Members:**
1. Go to **My Team** page
2. View all members and their roles
3. **Assign roles:**
   - Frontend developer
   - Backend developer
   - Designer
   - Tester

**Handling Join Requests:**
1. Check **Pending Requests** section
2. Review applicant profile
3. Accept or reject with message
4. New members notified automatically

**Removing Members:**
- Only in special circumstances
- Requires valid reason
- Member receives notification

### 2. Project Planning

**Creating Milestones:**
1. Go to **Timeline** page
2. Click **Add Milestone**
3. Set milestone details and deadline
4. Assign tasks to milestone
5. Track progress

**Task Assignment Strategy:**
- Distribute based on skills
- Balance workload
- Set realistic deadlines
- Regular check-ins

### 3. Progress Tracking

**Weekly Reviews:**
1. Check dashboard analytics
2. Review completed vs planned tasks
3. Identify blockers
4. Adjust plan as needed

**Reporting to Supervisor:**
1. Prepare weekly progress report
2. Include:
   - Completed tasks
   - Challenges faced
   - Next week's plan
   - Team velocity
3. Submit in **Reports** section

### 4. Meetings & Communication

**Scheduling Team Meetings:**
1. Go to **Calendar**
2. Click **Schedule Meeting**
3. Set:
   - Date and time
   - Duration
   - Agenda
   - Required attendees
4. Send invitations

**Meeting Best Practices:**
- Prepare agenda in advance
- Take notes during meeting
- Record decisions
- Assign action items
- Follow up after meeting

## Doctor/Supervisor Guide

### 1. Managing Students

**Viewing Assigned Teams:**
1. Dashboard shows all your teams
2. Click team to view details
3. See team composition and progress

**Team Performance Overview:**
- Health indicators (healthy/at-risk/critical)
- Completion rates
- Submission history
- Meeting attendance

### 2. Reviewing Submissions

**Proposal Review:**
1. Go to **Proposals** page
2. Click proposal to review
3. Read all sections thoroughly
4. Provide feedback:
   - Section-by-section comments
   - Overall assessment
   - Required changes
5. Approve or request revisions

**Grading Submissions:**
1. Open submission from **Submissions**
2. Download and review files
3. Use rubric for grading:
   - Score each criterion
   - Add comments
   - Calculate total
4. Submit grade

### 3. Evaluation System

**Using Rubrics:**
1. Select appropriate rubric
2. Rubrics available for:
   - Proposals
   - Midterm presentations
   - Final projects
3. Each criterion has weight and max score

**Providing Feedback:**
- Be specific and constructive
- Highlight strengths
- Suggest improvements
- Link to resources if helpful

### 4. Monitoring Progress

**Risk Management:**
- Check team health dashboard
- Red flags:
  - Low completion rate (<60%)
  - Missed deadlines
  - Low meeting attendance
- Intervene early with at-risk teams

**Analytics:**
1. Go to **Analytics** page
2. View:
   - Team comparison charts
   - Individual student performance
   - Submission trends
   - Grade distributions

### 5. Office Hours & Support

**Setting Office Hours:**
1. Go to **Profile** → **Office Hours**
2. Set available time slots
3. Students can book appointments
4. Sync with your calendar

**Virtual Consultations:**
- Accept meeting requests
- Use integrated video calls
- Record important advice
- Follow up with email summary

### 1. System Configuration

**User Management:**
1. Go to **Admin** → **Users**
2. Actions available:
   - Add new users (bulk import)
   - Edit user details
   - Assign roles
   - Reset passwords
   - Deactivate accounts

**Role Permissions:**
- **Admin**: Full system access
- **Doctor**: Team supervision & grading
- **TA**: Team support & code review
- **Leader**: Team management
- **Member**: Task execution

### 2. Academic Year Setup

**Semester Configuration:**
1. Go to **Admin** → **Settings**
2. Set:
   - Semester start/end dates
   - Key milestones
   - Submission deadlines
   - Evaluation periods

**Team Formation Period:**
- Set opening date for team creation
- Set deadline for team finalization
- Configure team size limits

### Enabling Notifications
1. **Settings** → **Notifications**
2. Toggle notifications for:
   - Tasks
   - Meetings
   - Submissions
   - Messages

4. Save preferences

1. **Settings** → **Appearance**
2. Select theme:
   - Light mode
   - Dark mode
   - Auto (follows system)

4. Preview before applying

   - Tasks
   - Messages
   - Submissions
   - Activity history
3. Click **Request Export**

## Troubleshooting

### Can't Login
- Check email and password
- Try password reset
- Clear browser cache
- Contact admin if persists

### Not Receiving Notifications
- Check notification settings
- Verify email address
- Check spam folder
- Test with a known action

### File Upload Fails
- Check file size (<10MB for most)
- Verify file format allowed
- Check internet connection
- Try different browser

### Calendar Not Syncing
- Reconnect calendar integration
- Check authorization hasn't expired
- Refresh calendar manually
- Contact support if continues

## Keyboard Shortcuts

### Global
- `Ctrl + K`: Quick search
- `Ctrl + /`: Show shortcuts
- `Esc`: Close dialog
- `Ctrl + B`: Toggle sidebar

### Navigation
- `G then D`: Go to Dashboard
- `G then T`: Go to Tasks
- `G then C`: Go to Calendar
- `G then M`: Go to Messages

### Tasks
- `N`: New task
- `E`: Edit selected task
- `D`: Delete selected task
- `Enter`: Open task details
