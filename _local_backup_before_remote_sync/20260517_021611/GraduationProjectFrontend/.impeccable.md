## Design Context

### Users
ProjectHub is used by graduation project students, team leaders, doctors, teaching assistants, and admins in an academic workflow. They need to understand project status quickly, coordinate meetings, track deadlines, review work, and move between team tasks without visual noise.

### Brand Personality
Clear, academic, capable. The interface should feel modern and calm, with enough polish to build confidence but enough restraint for repeated daily use.

### Aesthetic Direction
Operational dashboard UI: light, organized, responsive, and scan-friendly. Use purposeful color for state, priority, and action. Avoid decorative hero layouts, heavy glass effects, crowded card grids, and sections that only repeat information.

### Design Principles
- Put the next useful action and upcoming work first.
- Prefer clean grouped information over repeated cards and decorative metrics.
- Keep controls obvious for all roles: students, leaders, supervisors, and admins.
- Use motion to clarify state changes and page hierarchy, respecting reduced-motion users.
- Design mobile as a first-class workflow, not a shrunken desktop.
